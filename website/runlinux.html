Getting Started on Linux

System requirements: Linux 2.6.23 or later
Processor Architecture: x86 64-bit(amd64) 32-bit(386)

Download and Install PubSubSQL

PubSubSQL 64-bit distribution pubsubsql-v1.0.0-linux-amd64.tar.gz

PubSubSQL 32-bit distribution pubsubsql-v1.0.0-linux-386.tar.gz

Download one of the above archives and extract it into /usr/local, creating a PubSubSQL tree in /usr/local/pubsubsql. For example: 

$ sudo tar -C /usr/local -xzf pubsubsql-v1.0.0-linux-amd64.tar.gz

To be able to run server and client from the command line without changing to pubsubsql/bin directory add /usr/local/pubsubsql/bin to the PATH environment variable. Do this by adding this line to the end of your $HOME/.bashrc file:

export PATH=$PATH:/usr/local/pubsubsql/bin

Make sure to restart your terminal for change to the PATH to take effect.

Run PubSubSQL server from the command line
$ pubsubsql
info: 2014/02/11 18:00:00 started

To quit type letter q and press enter.

q
info: 2014/02/11 18:01:00 stoped

By default PubSubSQL server listens on all available network interfaces and binds to the default port:7777. To start PubSubSQL server listening on host port specify -port and/or -ip options. For example:

$ pubsubsql -port=1234 -ip=192.168.1.1
info: 2014/02/11 18:01:00 started
Intall and run PubSubSQL server as a service

If you want to deploy PubSubSQL server you should run the server as a linux service. Install PubSubSQL server as a service using service installer included in the distribution. Same -ip and -port options can be specified when installing the service. The service installer must be run as root or through sudo.

To install the service:

$ cd /usr/local/pubsubsql/bin 
$ sudo ./pubsubsqlsvc install

To uninstall the service:

$ cd /usr/local/pubsubsql/bin 
$ sudo ./pubsubsqlsvc uninstall

To start the service:

$ service pubsubsqld start

To stop the service:

$ service pubsubsqld stop

When ran as a service all output is redirected to syslog.

Run PubSubSQL client from the command line

Built in command line utility (cli) is provided that ca be used to send commands to PubSubSQL server. Run cli and send status command to the server:

$ pubsubsql cli
pubsubsql>status
{"status":"ok","action":"status","connections":1}
pubsubsql>
Run PubSubSQL Interactive Query gui client

PubSubSQL Interactive Query java gui client is provided that can be used to send command to PubSubSQL server. It displays the results in the grid view.

To run the gui: 

$ java -jar /usr/local/pubsubsql/bin/pubsubsqlgui.jar
