Download and install the distribution

PubSubSQL x86 64-bit distribution: pubsubsql-v1.0.0-linux-x64.tar.gz

PubSubSQL x86 32-bit distribution: pubsubsql-v1.0.0-linux-x86.tar.gz

Download one of the above archives and extract it into /usr/local, creating a PubSubSQL tree in /usr/local/pubsubsql. For example: 

$ sudo tar -C /usr/local -xzf pubsubsql-v1.0.0-linux-amd64.tar.gz

To be able to run the server and the CLI from the command line without changing to the /usr/local/ubsubsql/bin directory, add /usr/local/pubsubsql/bin to the PATH environment variable. For example, you can do this by adding the following line to the end of your ~/.bashrc file:

# add to the end of ~/.bashrc file
export PATH=$PATH:/usr/local/pubsubsql/bin

Run the following command for the changes to PATH to take effect:

$ source ~/.bashrc
Test your installation

Run the server:

$ pubsubsql
info: 2014/02/11 18:00:00 started

Quit the server by typing the letter q and press enter.

q
info: 2014/02/11 18:01:00 stopped

By default, the server listens for incoming connections on all available network interfaces and binds to the default port:7777. You may choose to specify a different port or host by using the -port and/or -ip options. For example:

$ pubsubsql -port=1234 -ip=192.168.1.1
info: 2014/02/11 18:01:00 started

CLI, the Command Line utility, is built into the pubsubsql executable. It can be used to send commands to the server. Run CLI and send a status command to the server:

$ pubsubsql cli
pubsubsql>status
{"status":"ok","action":"status","connections":1}
pubsubsql>
Configure and run as a daemon (service)

If you want to deploy the PubSubSQL server, you should run the server as a linux daemon (service). Install the server as a service using the service installer included in the distribution. The same -ip and -port options can be specified when installing the service. The service installer must be run as root or through sudo.

Install the service:

$ cd /usr/local/pubsubsql/bin 
$ sudo ./pubsubsqlsvc install

Uninstall the service:

$ cd /usr/local/pubsubsql/bin 
$ sudo ./pubsubsqlsvc uninstall

Start the service:

$ service pubsubsqld start

Stop the service:

$ service pubsubsqld stop

When the server is run as a service, the output is redirected to syslog.

Run PubSubSQL Interactive Query gui client

The PubSubSQL Interactive Query gui is provided to be used to send commands to the PubSubSQL server. It displays the results in a grid view as shown below.

Run the gui: 

$ java -jar /usr/local/pubsubsql/bin/pubsubsqlgui.jar
